# **Resumen Tema 4.** <br> Usabilidad

## 1. Concepto de Usabilidad

Disciplina que estudia cómo diseñar interfaces para una interacción eficiente.

### Diferencias

| Aspecto        | Descripción                                                                 |
|--------------------|-----------------------------------------------------------------------------|
| **Accesibilidad**  | Diseño inclusivo para personas con discapacidades.                          |
| **Utilidad**       | Grado en que el software satisface necesidades.                             |

### Características de una 'Interfaz Usable'

Útil, fácil de usar, fácil de aprender, elegante en diseño.

### Atributos de la Usabilidad

| Atributo           | Descripción                                                                 |
|------------------------|-----------------------------------------------------------------------------|
| **Facilidad de aprendizaje** | Fácil de aprender para nuevos usuarios.                                     |
| **Eficiencia**         | Permite a los usuarios realizar tareas rápidamente.                         |
| **Recuerdo de uso**    | Fácil de recordar después de un tiempo sin usarla.                          |
| **Tasas de error bajas** | Minimiza errores durante el uso.                                           |
| **Satisfacción**       | Los usuarios se sienten cómodos y satisfechos al usarla.                    |

## 2. Medida de Usabilidad

### Métricas

| Métrica       | Descripción                                                                 |
|-------------------|-----------------------------------------------------------------------------|
| **Efectividad**   | Tareas completadas, éxito en el primer intento.                             |
| **Eficiencia**    | Tiempo empleado, errores cometidos, clics necesarios.                       |
| **Satisfacción**  | Recomendaciones, calificaciones de facilidad de uso.                        |

### Proceso

Definir tareas, establecer objetivos, seleccionar variables, recoger datos.

## 3. Pruebas de Expertos

### Tipos

| Tipo                       | Descripción                                                                 |
|--------------------------------|-----------------------------------------------------------------------------|
| **Evaluación heurística**       | Evaluación basada en principios de usabilidad.                              |
| **Revisión de normas**          | Verificación del cumplimiento de estándares.                                |
| **Inspección de consistencia**  | Revisión de la coherencia en el diseño.                                     |
| **Inspección formal de usabilidad** | Evaluación detallada de la interfaz.                                     |

### Técnicas

| Técnica           | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Caminata cognitiva** | Simulación del uso de la interfaz por expertos.                             |

## 4. Pruebas con Usuarios

### Metodologías

| Metodología       | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Reuniones**         | Discusión con usuarios para obtener feedback.                               |
| **Entrevistas**       | Preguntas directas para entender necesidades.                               |
| **Encuestas**         | Cuestionarios para recopilar opiniones.                                     |
| **Diseño de escenarios** | Creación de situaciones de uso para evaluar la interfaz.                  |
| **Diseño participativo** | Involucrar a los usuarios en el diseño de la interfaz.                    |

### Muestra

5-10 usuarios suelen ser suficientes para detectar problemas.

## 5. Pautas de Diseño de la Interfaz

### Principios

| Principio       | Descripción                                                                 |
|---------------------|-----------------------------------------------------------------------------|
| **Organización**    | Estructura clara y lógica de la interfaz.                                   |
| **Economía**        | Minimizar el esfuerzo del usuario para realizar tareas.                     |
| **Comunicación**    | Transmitir información de manera clara y eficiente.                         |

### Aspectos Específicos

| Aspecto         | Descripción                                                                 |
|---------------------|-----------------------------------------------------------------------------|
| **Diseño visual**   | Estética y disposición de elementos.                                        |
| **Color**           | Uso adecuado de colores para resaltar información.                          |
| **Matiz**           | Variaciones de color para diferenciar elementos.                            |
| **Contraste**       | Diferenciación clara entre elementos.                                       |
| **Disposición de elementos** | Organización lógica y coherente.                                     |
| **Redacción de texto** | Claridad y concisión en los textos.                                      |

## 6. Estructura de la Interfaz

### Principios

| Principio           | Descripción                                                                 |
|-------------------------|-----------------------------------------------------------------------------|
| **Familiaridad del usuario** | Diseño que se ajusta a las expectativas del usuario.                     |
| **Consistencia**        | Coherencia en el diseño y comportamiento de la interfaz.                  |
| **Legibilidad**         | Facilidad para leer y entender la información.                             |
| **Mínima sorpresa**     | Evitar comportamientos inesperados.                                        |
| **Recuperabilidad**     | Permitir al usuario corregir errores fácilmente.                           |

### Elementos

| Elemento           | Descripción                                                                 |
|------------------------|-----------------------------------------------------------------------------|
| **Menús**              | Barras, desplegables, paletas, contextuales, personalización.              |
| **Ventanas**           | Áreas dedicadas a tareas específicas.                                      |
| **Cuadros de Diálogo** | Información de éxito/fracaso, motores de búsqueda, ayuda en línea.         |
| **Atajos de Teclado**  | Combinaciones de teclas para acciones frecuentes.                          |

## 7. Aspecto de la Interfaz

### Colores

| Aspecto           | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Ventajas**          | Comunicación rápida, resaltar información, impacto psicológico.             |
| **Precauciones**      | Evitar abuso, limitar número de colores, consistencia.                     |

### Fuentes

| Aspecto           | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Legibilidad**       | Facilidad para leer el texto.                                               |
| **Tamaño adecuado**   | Tamaño de fuente que facilite la lectura.                                   |
| **Evitar bloques extensos** | No usar párrafos demasiado largos.                                      |

### Iconos

| Aspecto           | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Reconocimiento rápido** | Iconos que se entiendan de inmediato.                                     |
| **Equilibrio con texto** | Combinación adecuada de iconos y texto.                                   |
| **Evitar sobrecarga gráfica** | No saturar la interfaz con demasiados iconos.                          |

### Distribución de Elementos

| Aspecto           | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Organización clara** | Distribución lógica y coherente de los elementos.                           |
| **Uso de retículas**   | Alineación de elementos usando retículas.                                   |
| **Espacios en blanco** | Uso adecuado de espacios para evitar saturación visual.                     |

## 8. Elementos Interactivos

### Botones de Comando

| Aspecto           | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Tipos**             | En relieve, confirmación, radio.                                           |
| **Consideraciones**   | Intuitivos, ubicación accesible.                                           |

### Listas Desplegables

| Aspecto           | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Utilidad**          | Útiles para selección de opciones, autocompletar.                           |

### Etiquetas

| Aspecto           | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Función**           | Textos estáticos que identifican elementos.                                 |

## 9. Presentación de Datos

### Reglas de Diseño

| Regla             | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Enrejillado**       | Organización en cuadrícula para alinear elementos.                          |
| **Balanceado**        | Distribución equilibrada de elementos.                                      |
| **Simetría**          | Diseño simétrico para una apariencia ordenada.                              |

### Consejos

| Consejo           | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Ser conciso**       | Evitar información innecesaria.                                             |
| **Facilitar navegación** | Hacer que la navegación sea intuitiva.                                    |
| **Estructura piramidal** | Organizar la información de lo general a lo específico.                   |
| **Lenguaje cercano**  | Usar un lenguaje que el usuario entienda fácilmente.                        |
| **Información separada** | Dividir la información en secciones claras.                              |

## 10. Diseño de la Secuencia de Control

### Proceso

| Paso              | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Creación de la interfaz** | Diseño inicial de la interfaz.                                            |
| **Establecimiento de propiedades** | Definir las características de los elementos.                          |
| **Escritura de código** | Implementación de la interfaz.                                            |
| **Pruebas**           | Verificación del funcionamiento de la interfaz.                             |
| **Generación del ejecutable** | Creación del archivo final para su distribución.                        |

### Consideraciones

| Consideración     | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Definir objetivos** | Establecer claramente los objetivos de la interfaz.                          |
| **Asociar interacciones** | Relacionar las acciones del usuario con las respuestas de la interfaz.     |
| **Especificar acciones** | Definir cómo se deben realizar las acciones.                               |
| **Asegurar interpretación correcta** | Garantizar que el usuario entienda cómo usar la interfaz.               |

## 11. Aseguramiento de la Información

### Ámbitos

| **Ámbito**            | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Datos**             | Protección de la información almacenada.                                    |
| **Procesos**          | Seguridad en los procesos de la aplicación.                                 |
| **Comportamiento**    | Control de las acciones del usuario.                                        |
| **Sistema de gestión** | Administración segura del sistema.                                         |

### Herramientas de Seguridad

| **Herramienta**       | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Control de acceso** | Restricción de acceso a ciertas partes del sistema.                          |
| **Políticas de seguridad** | Normas para proteger la información.                                     |
| **Criptografía**      | Uso de técnicas de cifrado para proteger datos.                             |
| **Certificados y claves** | Autenticación y encriptación de datos.                                  |

## 12. Aplicaciones Multimedia

### Elementos

| **Elemento**          | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Gráficos**          | Imágenes y diseños visuales.                                                |
| **Imágenes**          | Fotografías y otros elementos visuales.                                     |
| **Animaciones**       | Movimiento de elementos para captar la atención.                            |
| **Vídeos**            | Contenido audiovisual.                                                      |
| **Audio**             | Sonidos y música.                                                           |

### Consideraciones

| **Consideración**     | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Equilibrio entre multimedia y texto** | Combinar adecuadamente ambos elementos.                              |
| **Evitar exceso**     | No saturar la interfaz con demasiados elementos multimedia.                  |
| **Coherencia en contenidos** | Asegurar que los elementos multimedia sean relevantes y coherentes.      |

## 13. Resumen final

| Aspecto           | Descripción                                                                 |
|-----------------------|-----------------------------------------------------------------------------|
| **Usabilidad**        | Clave para el éxito de una aplicación, asegurando que sea fácil de usar, aprender y recordar. |
| **Pruebas con expertos y usuarios** | Ayudan a detectar problemas de diseño.                                  |
| **Diseño de la interfaz** | Debe ser claro, consistente y equilibrado, con un uso adecuado de colores, fuentes y elementos interactivos. |
| **Presentación de datos** | Debe ser concisa y facilitar la navegación.                                |
| **Aseguramiento de la información** | Esencial para una experiencia de usuario satisfactoria.                  |
| **Uso equilibrado de elementos multimedia** | Mejora la experiencia sin saturar la interfaz.                        |
